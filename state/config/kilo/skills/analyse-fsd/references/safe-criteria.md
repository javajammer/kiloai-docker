# SAFe (Scaled Agile Framework) Methodology Compliance Criteria (S1-S8)

Use this checklist when the detected methodology is SAFe.
Evaluate each criterion and assign PASS / PARTIAL / FAIL status.

---

## S1: PI Planning

| Field | Content |
|-------|---------|
| **ID** | S1 |
| **Area** | PI Planning |
| **Check** | Are features aligned to PI (Program Increment) objectives with clear business value? |
| **Evidence to Look For** | PI objectives document, feature-to-objective mapping, business value scoring, PI planning event records |
| **Severity if Missing** | CRITICAL |
| **PASS Criteria** | PI planning conducted on regular cadence (8-12 weeks). Features ranked by WSJF or similar. PI objectives have business value assigned. Program board shows dependencies and milestones. |
| **PARTIAL Criteria** | PI planning exists but features not aligned to objectives, or business value not scored. |
| **FAIL Criteria** | No PI planning. Features are developed without program-level alignment. |

---

## S2: ART (Agile Release Train) Readiness

| Field | Content |
|-------|---------|
| **ID** | S2 |
| **Area** | ART Readiness |
| **Check** | Is the Agile Release Train prepared with all teams, scrum masters, product owners, and release train engineer? |
| **Evidence to Look For** | ART roster, team structure, RTE role defined, PO/SM identified per team, ART sync cadence |
| **Severity if Missing** | HIGH |
| **PASS Criteria** | ART fully staffed with all teams. Roles filled: RTE, Product Owners, Scrum Masters. ART ceremonies defined (ART sync,Inspect & Adapt, PI Planning). All teams on same cadence. |
| **PARTIAL Criteria** | ART exists but some teams lack POs/SMs, or teams are on different cadences. |
| **FAIL Criteria** | No clear ART structure. Teams operate independently without program-level coordination. |

---

## S3: Cross-ART Dependencies

| Field | Content |
|-------|---------|
| **ID** | S3 |
| **Area** | Dependency Management |
| **Check** | Are cross-ART dependencies identified, mapped, and managed proactively? |
| **Evidence to Look For** | Dependency board, program board with dependency lines, inter-ART coordination, risk/dependency management |
| **Severity if Missing** | HIGH |
| **PASS Criteria** | Dependencies visualized on program board. Cross-ART dependencies have owners and timelines. Risk mitigation plans for critical dependencies. Regular dependency review in PI planning. |
| **PARTIAL Criteria** | Dependencies identified but not tracked, or no mitigation plans for high-risk dependencies. |
| **FAIL Criteria** | Dependencies not mapped. Teams discover blockers late in sprint/PI. |

---

## S4: Lean Portfolio Management

| Field | Content |
|-------|---------|
| **ID** | S4 |
| **Area** | Lean Portfolio |
| **Check** | Is there portfolio-level governance with lean budgeting and strategic alignment? |
| **Evidence to Look For** | Portfolio kanban, lean budget accounts, strategic themes, portfolio sync, Epic owners |
| **Severity if Missing** | HIGH |
| **PASS Criteria** | Portfolio kanban manages epics from ideation to implementation. Lean budget accounts allocated to value streams. Strategic themes link portfolio to strategy. Portfolio sync cadence established. Epic owners identified. |
| **PARTIAL Criteria** | Some portfolio governance but incomplete (e.g., no portfolio kanban, or no lean budgets). |
| **FAIL Criteria** | No portfolio-level governance. Investment decisions made ad-hoc without strategic alignment. |

---

## S5: SAFe Metrics & Predictability

| Field | Content |
|-------|---------|
| **ID** | S5 |
| **Area** | Metrics |
| **Check** | Are SAFe metrics (PI predictability, WSJF, innovation accounting) tracked and used? |
| **Evidence to Look For** | PI predictability chart, WSJF scores, innovation metrics, team-level metrics, improvement trends |
| **Severity if Missing** | MEDIUM |
| **PASS Criteria** | PI predictability tracked (target: 80-100%). WSJF used for feature prioritization. Team metrics visible (velocity, defect rate, cycle time). Innovation accounting for new ventures. |
| **PARTIAL Criteria** | Some metrics tracked but not used for decision-making, or key metrics missing. |
| **FAIL Criteria** | No metrics. Decisions based on gut feel rather than data. |

---

## S6: Innovation & Technical Excellence

| Field | Content |
|-------|---------|
| **ID** | S6 |
| **Area** | Innovation |
| **Check** | Is there allocated capacity for innovation, exploration, and technical excellence? |
| **Evidence to Look For** | Innovation sprints, Hackathon/Innovation & Adaptation, tech debt allocation, Enabler features |
| **Severity if Missing** | MEDIUM |
| **PASS Criteria** | Capacity allocated for innovation (e.g., innovation sprints, Hackathons). Enabler features in backlog for technical excellence. Team has autonomy to experiment. Innovation budget protected from feature pressure. |
| **PARTIAL Criteria** | Innovation mentioned but no dedicated capacity or budget. |
| **FAIL Criteria** | 100% feature focus. No capacity for innovation, learning, or technical improvement. |

---

## S7: Lean Budgeting

| Field | Content |
|-------|---------|
| **ID** | S7 |
| **Area** | Lean Budget |
| **Check** | Are budgets allocated to value streams (not projects) with guardrails for spending? |
| **Evidence to Look For** | Value stream budget allocation, spending guardrails, portfolio capacity allocation, investment horizons |
| **Severity if Missing** | HIGH |
| **PASS Criteria** | Budgets assigned to value streams, not individual projects. Guardrails define spending limits. Investment horizons balanced (Horizon 1: sustain, Horizon 2: expand, Horizon 3: explore). Budget flexibility for adaptive planning. |
| **PARTIAL Criteria** | Some lean budgeting but still project-based funding, or no guardrails. |
| **FAIL Criteria** | Traditional project-based budgeting. No value stream alignment. |

---

## S8: Strategic Themes Alignment

| Field | Content |
|-------|---------|
| **ID** | S8 |
| **Area** | Strategic Themes |
| **Check** | Do initiatives link to strategic themes with clear business outcomes? |
| **Evidence to Look For** | Strategic themes document, Epic-to-theme mapping, business outcomes, strategic review cadence |
| **Severity if Missing** | MEDIUM |
| **PASS Criteria** | Strategic themes defined by portfolio leadership. Epics and features trace to strategic themes. Business outcomes measured. Regular strategic review and adjustment. |
| **PARTIAL Criteria** | Strategic themes exist but not linked to portfolio work, or outcomes not measured. |
| **FAIL Criteria** | No strategic themes. Portfolio work disconnected from business strategy. |

---

## Scoring Guide

| Score | Criteria |
|-------|----------|
| **8/8 PASS** | Fully compliant — SAFe implementation is mature and effective |
| **6-7 PASS** | Mostly compliant — Minor gaps, acceptable with improvement plan |
| **4-5 PASS** | Partially compliant — Significant gaps, SAFe adoption needs coaching |
| **<4 PASS** | Non-compliant — "SAFe" in name only, or still transitioning from another methodology |
