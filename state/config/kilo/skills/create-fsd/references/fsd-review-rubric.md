# FSD (Functional Specification Design) Review Rubric

## Expected Sections (checklist)

- [ ] Project Background & Objectives
- [ ] Project Scope (In-Scope / Out-of-Scope)
- [ ] Stakeholder Identification
- [ ] Functional Requirements
- [ ] Non-Functional Requirements
- [ ] Data Architecture / Data Model
- [ ] System Architecture / Integration Design
- [ ] UI/UX Specifications or Mockups
- [ ] Business Rules & Logic
- [ ] Security Requirements (RLS, RBAC, Encryption)
- [ ] Testing Strategy & Acceptance Criteria
- [ ] Implementation Timeline
- [ ] Dependencies & Assumptions
- [ ] Risk Register
- [ ] Change Management Plan
- [ ] Handover & Support Plan
- [ ] Sign-off / Approval Matrix

## Critical Questions

1. Are "Done" criteria specific and measurable?
2. Who owns the system after go-live? Is this explicit?
3. Are data ownership and portability rights defined?
4. Is the monitoring strategy sufficient (not just email alerts)?
5. What is the SLA for incident response?
6. Are rollback and disaster recovery plans documented?
7. Is the testing strategy realistic with clear pass/fail criteria?
8. Does the scope management process handle change requests fairly?
