# Methodology Compliance Matrix

## Agile/Scrum Review Criteria

| Area | Check | Severity if Missing |
|------|-------|-------------------|
| Sprint Planning | Are sprints time-boxed with clear goals? | HIGH |
| Backlog | Is backlog prioritized and estimated? | HIGH |
| User Stories | Do stories follow INVEST criteria? | MEDIUM |
| Definition of Done | Is DoD explicitly defined? | HIGH |
| Technical Debt | Is there a budget for tech debt per sprint? | MEDIUM |
| Retrospectives | Are retro findings tracked and acted upon? | MEDIUM |
| Velocity | Is velocity tracked and used for forecasting? | LOW |
| Cross-functional | Does the team have all needed skills? | HIGH |

## SAFe (Scaled Agile Framework) Review Criteria

| Area | Check | Severity if Missing |
|------|-------|-------------------|
| PI Planning | Are features aligned to PI objectives? | CRITICAL |
| ART Readiness | Is the Agile Release Train prepared? | HIGH |
| Dependencies | Are cross-ART dependencies mapped? | HIGH |
| Lean Portfolio | Is there portfolio-level governance? | HIGH |
| Metrics | Are SAFe metrics (WSJF, predictability) tracked? | MEDIUM |
| Innovation | Is there capacity for innovation sprints? | MEDIUM |
| Lean Budget | Are budgets allocated to value streams? | HIGH |
| Strategic Themes | Do initiatives link to strategic themes? | MEDIUM |

## Waterfall Review Criteria

| Area | Check | Severity if Missing |
|------|-------|-------------------|
| Requirements Phase | Are requirements baselined and signed off? | CRITICAL |
| Design Phase | Is design reviewed before implementation? | CRITICAL |
| Traceability | Is there a requirements traceability matrix? | HIGH |
| Change Control | Is there a formal change request process? | HIGH |
| Phase Gates | Are phase gate criteria defined? | HIGH |
| V-Model | Are verification steps mapped to requirements? | MEDIUM |
| Testing Plan | Is testing planned per phase (unit/integration/UAT)? | HIGH |
| Documentation | Are deliverables per phase documented? | MEDIUM |
